## Project Overview
This project analyzes telecom customer data to identify customer behavior patterns, revenue trends, recharge activity, and ARPU (Average Revenue Per User) performance. The analysis was performed using Excel, SQL, Python, and Power BI to transform raw telecom data into actionable business insights and an interactive dashboard.

The project demonstrates an end-to-end analytics workflow, including data cleaning, KPI analysis, SQL-based business queries, Python exploratory data analysis, and Power BI dashboard development.

## Business Problem
Telecom companies manage millions of customer records and need data-driven insights to improve customer retention, maximize revenue, and understand customer usage patterns. However, large datasets often make it difficult to identify key trends and customer segments.

This project aims to analyze customer behavior, recharge patterns, and revenue performance to support better business decision-making.

## Project Objectives

1. Analyze telecom customer behavior and revenue trends
2. Measure key business KPIs such as Total Customers, Total Revenue, Average ARPU, and Average Recharge Amount
3. Segment customers based on ARPU and recharge behavior
4. Identify high-value and low-value customer groups
5. Perform data analysis using SQL and Python
6. Develop an interactive Power BI dashboard for business reporting
7. Generate actionable insights for telecom business growth

## Dataset Information
**Dataset Size:** Approximately 100,000 customer records

The dataset contains customer-level information including:

1. Customer ID
2. ARPU (Average Revenue Per User)
3. Recharge Amount
4. Customer Revenue
5. Recharge Categories

7. Customer Segmentation Variables
